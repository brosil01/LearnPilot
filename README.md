# LearnPilot Backend

FastAPI backend powering LearnPilot's two AI-driven learning modes using **Qwen2.5 via Groq**.

## Architecture

```
Frontend (React)
    ↓ POST /generate
FastAPI (main.py)
    ↓
AIEngine (ai_engine.py)   ← Groq API → Qwen2.5-32B
    ↓
HistoryService + SystemMonitor
```

## Quick Start

### 1. Get a free Groq API key
Go to [console.groq.com](https://console.groq.com) → Sign up → Create API Key (free)

### 2. Set up environment
```bash
cp .env.example .env
# Edit .env and paste your GROQ_API_KEY
```

### 3. Install dependencies
```bash
python -m venv venv
source venv/bin/activate        # Mac/Linux
# OR
source venv/Scripts/activate    # Windows

pip install -r requirements.txt
```

### 4. Run the server
```bash
uvicorn main:app --reload
```

Server runs at: `http://127.0.0.1:8000`
API docs at: `http://127.0.0.1:8000/docs` (auto-generated by FastAPI)

### 5. Test it
```bash
python test_api.py
```

## API Contract

### POST `/generate`
**Request:**
```json
{
  "topic": "Binary Search",
  "mode": "Concept-First Learning",
  "code_snippet": "optional code string"
}
```

**Modes:**
- `"Concept-First Learning"` — builds from fundamentals up
- `"Reverse Engineering"` — starts with full solution, deconstructs

**Response:**
```json
{
  "mode": "Concept-First Learning",
  "explanation": "summary text",
  "raw": "full markdown response — render this on frontend",
  "steps": ["Step 1", "Step 2", "Step 3"],
  "exercise": "Practice question text"
}
```

### GET `/history`
Returns all past sessions stored in `history.json`.

### GET `/health`
Returns `{"status": "ok"}` — use for deployment health checks.

## Swapping Models

Change one line in `.env`:
```
MODEL_ID=qwen-qwq-32b              # Best quality (default)
MODEL_ID=llama-3.3-70b-versatile   # LLaMA alternative
MODEL_ID=llama3-8b-8192            # Fastest/lightest
```

## Frontend Integration (for Kashif & Priyanshu)

```javascript
const response = await fetch('http://127.0.0.1:8000/generate', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        topic: "Binary Search",
        mode: "Concept-First Learning",  // or "Reverse Engineering"
        code_snippet: null               // optional
    })
});

const data = await response.json();

// Use data.raw for full markdown rendering (recommended)
// Use data.steps for sidebar/progress display
// Use data.exercise for the practice question section
console.log(data.raw);
```

**Render `data.raw` with a markdown library** like `react-markdown` for best results.

## Deployment

### Railway (recommended)
1. Push to GitHub
2. Connect repo to [railway.app](https://railway.app)
3. Add `GROQ_API_KEY` and `MODEL_ID` as environment variables
4. Deploy — Railway auto-detects FastAPI
